        Transformations done:
        1) column names get standardized  
                -Every source system has its own naming quirks. Staging makes them consistent.
        2) types get fixed  
                -Raw data often has strings where integers should be, timestamps in weird formats, booleans stored as text, etc.

        3) JSON gets flattened  
                -This is especially important for your MongoDB events. Staging is where nested structures become clean relational columns.
        4) weird source quirks get normalized  
                -Examples:
                        -inconsistent casing
                        -null vs empty string
                        -inconsistent date formats
                        -inconsistent event names
                        -missing IDs
                        -duplicate rows
        5) naming conventions get applied  
                -This is where you enforce your project‑wide standards:
                -snake_case
                -id suffixes
                -created_at / updated_at
                -*_amount, *_count, etc.
